11/04/23 - Tables

To display tabular data, such as you might find in an excel spreadsheet or Pandas dataframe, in HTML, we (shockingly) use a table.

Enclosed within the <table> tag, we define headers with the table header tag <th>, ordinary rows with table rows <tr>, and cell values with <td> (table data).

Change PCT Change

S&P 500 1,305.75 -13.93 -1.06%
NASDAQ 2,731.64 -33.01 -1.19%

A couple of things to to note here. First, table data <td> is nested within each table row <tr>, which is nested with the table tags. And second, our top row is defined not with table data tags, but with table header tags <th>. Also, notice that there are two empty header tags. As headers are aligned with columns, we need those there to make sure the change and PCT change headers are place over the appropriate columns.

11/02/23 - Audio and Video in HTML5

One of the main topics covered this week is the addition of audio and video files into web pages. As common as multimedia is on the modern web, it's not as simple as adding an <img> tag for images. First, not all browsers support the same codecs, so it becomes necessary to provide multiple file types(e.g. having a video file provided in both .mp4 and .webm). Additionally, since some older browsers don't support HTML5, which introduces native multimedia, we must also add an error message for this scenario. Note also the attribute used: controls tells the browser to render the controls for the file. Control buttons are not displayed by default, so this attribute is necessary for the user to be able to play or stop the playback. Other attributes include:

  • autoplay - applies to both video an audio and indicates that the file shoule start playing automatically.
  • preload - applies to both video and audio and inidcates what the browser should do if autoplay is not enabled. It can take the following values:
    • none - browser should not load the file until the user presses play
    • auto - browser downloads the video when the page loads
    • metadata - browser collects metadata like size, duration and track list
  • width and height - determines the size that a video is rendered in the browser.
